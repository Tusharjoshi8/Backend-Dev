
PRACTICE QUESTIONS


Exercise 1: Basic CRUD (Student Management System)
1. Add New Student
db.students.insertOne({
  name: "Aman",
  email: "aman@example.com",
  gpa: 3.4,
  city: "Meerut",
  courses: ["DBMS", "OS", "CN"]
})
2. View All Students
db.students.find()
3. Find Student by Email
db.students.findOne({ email: "aman@example.com" })
4. Update Student GPA
db.students.updateOne(
  { email: "aman@example.com" },
  { $set: { gpa: 3.9 } }
)
5. Delete Student
db.students.deleteOne({ email: "aman@example.com" })


Exercise 2: Advanced Queries
1. GPA Between 3.0 and 3.5
db.students.find({
  gpa: { $gte: 3.0, $lte: 3.5 }
})
2. Students Enrolled in More Than 5 Courses
db.students.find({
  $expr: { $gt: [{ $size: "$courses" }, 5] }
})
3. Top 10 Students by GPA
db.students.find()
  .sort({ gpa: -1 })
  .limit(10)
4. Count Students by City
db.students.aggregate([
  {
    $group: {
      _id: "$city",
      totalStudents: { $sum: 1 }
    }
  }
])


Exercise 3: Schema Design (Mongoose)
1. Course Schema
const mongoose = require("mongoose");

const courseSchema = new mongoose.Schema({
  name: String,
  prerequisites: [{ type: String }]
});
2. Professor Schema
const professorSchema = new mongoose.Schema({
  name: String,
  departments: [{ type: String }]
});
3. Grade Schema
const gradeSchema = new mongoose.Schema({
  studentId: {
    type: mongoose.Schema.Types.ObjectId,
    ref: "Student"
  },
  courseId: {
    type: mongoose.Schema.Types.ObjectId,
    ref: "Course"
  },
  grade: String
});


Exercise 4: Aggregation Pipelines
1. Average GPA by Department
db.students.aggregate([
  {
    $group: {
      _id: "$department",
      avgGPA: { $avg: "$gpa" }
    }
  }
])
2. Most Popular Courses
db.students.aggregate([
  { $unwind: "$courses" },
  {
    $group: {
      _id: "$courses",
      count: { $sum: 1 }
    }
  },
  { $sort: { count: -1 } }
])
3. Student Performance Report
db.students.aggregate([
  {
    $project: {
      name: 1,
      gpa: 1,
      performance: {
        $cond: {
          if: { $gte: ["$gpa", 3.5] },
          then: "Excellent",
          else: "Needs Improvement"
        }
      }
    }
  }
])



ASSESSMENT QUESTIONS

1. Advantages of Mongoose over Native MongoDB Driver
Schema-based structure
Built-in validation
Middleware support
Easier relationship handling with populate
Cleaner and maintainable code.


2. 
findOneAndUpdate() updates a document and returns the document (either original or updated).
updateOne() updates a document but does not return the document, only the update result.
findOneAndUpdate() is used when you need the updated data immediately.
updateOne() is used for simple updates without needing the document back.
Both operations are atomic in MongoDB.


3. Purpose of Middleware in Mongoose
Middleware allows execution of functions before or after database operations.
Examples:
pre("save") for validation or hashing
post("find") for logging
Used for validation, logging, and transformation.


4. Pagination in Mongoose
const page = 1;
const limit = 10;
Student.find()
  .skip((page - 1) * limit)
  .limit(limit);


5. Embedding vs Referencing
Embedding :
Stores related data in a single document
Faster reads
Suitable for small and closely related data.

Referencing :
Uses ObjectId references
Avoids duplication
Suitable for large or reusable datasets.
